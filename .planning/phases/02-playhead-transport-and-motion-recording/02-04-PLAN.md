4. In the Scroll POS lane SVG, render the bezier curve path BEFORE the dots (so dots render on top). Compute a cubic bezier SVG `<path>` from the sorted keyframe array:
   - Sort `keyframeDots` by `position` ascending before use.
   - Build the path using smooth cubic bezier segments. For N keyframes, use a Catmull-Rom to cubic bezier conversion: for each pair of adjacent keyframes `[a, b]`, the control points can be approximated as one-third of the distance between neighbors. A simpler approach that works well: use `C` (cubic bezier) commands with control points pulled horizontally: for a segment from point `(x1, y1)` to `(x2, y2)`, use `C ${x1 + (x2-x1)/3} ${y1}, ${x2 - (x2-x1)/3} ${y2}, ${x2} ${y2}`.
   - The full path `d` attribute starts with `M x0 y0` and appends a `C` command for each subsequent keyframe.
   - Map keyframe coordinates: `cx = (kf.position / SEQUENCE_DURATION) * VB_W`, `cy = (1 - kf.value) * VB_H`.
   - Render as:
```tsx
{keyframeDots.length >= 2 && (() => {
  const sorted = [...keyframeDots].sort((a, b) => a.position - b.position);
  const pts = sorted.map(kf => ({
    x: (kf.position / SEQUENCE_DURATION) * VB_W,
    y: (1 - kf.value) * VB_H,
  }));
  let d = `M ${pts[0].x} ${pts[0].y}`;
  for (let i = 1; i < pts.length; i++) {
    const prev = pts[i - 1];
    const curr = pts[i];
    const dx = curr.x - prev.x;
    d += ` C ${prev.x + dx / 3} ${prev.y}, ${curr.x - dx / 3} ${curr.y}, ${curr.x} ${curr.y}`;
  }
  return (
    <path
      d={d}
      fill="none"
      stroke="#a855f7"
      strokeWidth="1.5"
      strokeOpacity="0.8"
    />
  );
})()}
```
